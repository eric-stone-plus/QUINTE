# 罗生门隐喻 — QUINTE 设计散文

> ⚠️ 此为设计灵感文档，非协议规范。QUINTE 的运作不依赖此隐喻。

---

## 隐喻根基

《罗生门》（1950）的核心不是"谁在说谎"，而是**单一视角永远不可信**。四个证人面对同一事件给出互相矛盾的供述——不是因为有人故意撒谎，而是因为每个人只看到自己能看的部分。

QUINTE 的架构设计来自同一个直觉：**单个 AI agent 的结论不可信，不是因为它在"骗你"，而是因为它只有一个视角。** 五方交叉对质不是为了找出"谁对"，而是让分歧和盲区在对比中自我暴露。

---

## 核心概念

### 罗生门门（Rashomon Gates）

SOUL.md 三道强制门的文化命名：

| # | 门 | 罗生门隐喻 | 机制 |
|---|-----|-----------|------|
| 1 | **雨門**（Ambiguity Gate） | 进入罗生门之前，确认你到底要审什么 | 模糊问题先 clarify，不猜 |
| 2 | **證門**（QUINTE Gate） | 多方供述，交叉对质 | R1+R2+R3 完整辩论 |
| 3 | **閂門**（Anti-Drift Gate） | 不许证人串供——每个人的"供述"必须独立 | 三层法防 agent 漂移 |

**命名逻辑**：借用罗生门城门的意象。雨門是城门入口（樵夫在雨中走进罗生门），證門是证人对质，閂門是门闩——防止外部关联污染证词。

### 罗生门深度（Rashomon Depth）

不是 agent 数量，而是**实际产生的独立视角数**。

```
Rashomon Depth = |{ genuinely different reasoning paths }|
```

- 同模型换 prompt 参数 → Depth = 1（换面具，不是新证人。POSTMORTEM #1）
- 不同模型、不同工具链、不同关注偏好 → Depth 增加
- R2 交叉审本身也贡献 Depth——换个角度看别人的分析

**经验法则**：如果四方 R1 输出惊人一致，且无人说"不确定"，**这可能是共享盲区，不是共识**。罗生门里如果四个人说的完全一样，故事就没意思了——现实中这通常意味着所有人都漏看了同一件事。

### 藪の中指数（Yabu no Naka Index）

R1 各方陈述的"不一致度"。取自芥川龙之介原著名《藪の中》（竹林中）。

```
YNI = 1 - (各方论点交集 / 各方论点并集)
```

- YNI ≈ 0：高度一致 → ⚠️ 可能共享盲区，R2 做确认性审计
- YNI ≈ 0.3-0.5：健康分歧 → R2 标注争议
- YNI > 0.7：高度分散 → 可能原问题本身模糊，退回雨門（Ambiguity Gate）

### 黑泽明检查（Kurosawa Check）

R2 强制交叉审——**罗生门的核心机制**。

黑泽明拍《罗生门》不只是让四个人各说各的。真正的力量在于：**观众在对比供述的过程中，自己推演出接近真相的理解。** R2 就是这个"对比"环节——每个 agent 审其他所有人的 R1，找出：

1. 四方一致但可能漏看的地方
2. 两方一致、一方异议的边界
3. 各方完全分歧的核心

**R2 永不跳过。** 就像你无法看完一个人的供述就下结论——必须看完所有人的，然后对照。

## 使用方式

这个框架是 QUINTE 的 **why** 层——不替代 PROTOCOL.md（how），而是解释为什么协议要这样设计。

- **写 QUINTE 终裁时**：使用这些概念描述结论。例如："藪の中指数 0.4，健康分歧；黑泽明检查确认 R2 三方一致"
- **向他人解释 QUINTE 时**：先讲罗生门隐喻，再讲协议细节
- **设计 QUINTE 改进时**：用框架概念作为判断标准——这个改动是否增加了罗生门深度？是否强化了黑泽明检查？

## 文化锚点

- 芥川龍之介《藪の中》（1922）
- 黒澤明《羅生門》（1950）— 威尼斯金狮奖，奥斯卡最佳外语片
- 罗生门效应（Rashomon Effect）— 已进入英语学术词汇，指同一事件的不同主观叙述
