# RASHOMON — QUINTE Design Philosophy

> 芥川龍之介《藪の中》（1922）→ 黒澤明《羅生門》（1950）→ QUINTE Protocol

## 一句话

**罗生门现象是问题。QUINTE 是解法。**

单一视角不可信 → 五方交叉对质 → 分歧在对比中自我暴露 → 收敛于最佳可用答案。

## 为什么叫这个名字

黑泽明的《罗生门》讲了四个人对同一事件给出互相矛盾的供述。不是因为有人撒谎——而是每个人只看到自己能看的部分。QUINTE 的架构设计来自同一个直觉：单个 AI agent 的结论不可信，不是因为它"骗你"，而是因为它只有一个视角。

## 在 QUINTE 中的位置

RASHOMON 是 QUINTE 的 **why** 层——不替代 [spec/PROTOCOL.md](../spec/PROTOCOL.md)（how），而是解释协议为什么这样设计。

- **PROTOCOL.md**: 协议规范（可执行规则）
- **RASHOMON/**: 设计哲学（为什么这些规则存在）
- **SOUL.md**: 运行时执行（三道门每 session 注入）

## 文化锚点

- 芥川龍之介《藪の中》（1922）— 七个证人对同一死亡事件的矛盾供述
- 黒澤明《羅生門》（1950）— 威尼斯金狮奖，奥斯卡最佳外语片，将"藪の中"嵌套进罗生门城门的框架叙事
- 罗生门效应（Rashomon Effect）— 已进入英语学术词汇，指同一事件的不同主观叙述
